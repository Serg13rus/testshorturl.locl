Для работы требуется php7.4 или выше.

Инструкция

1. Создать на сайте дерикторию (но можно использовать и корневую дерикторию).
2. Если используется корневая дериктория и там уже есть index.php, то можно либо переименовать index.php,
который находится в репозитарии, либо разместить html-код из него в существующем index.php и подключить script.js.
3. Если используется созданная дериктория, то поместить в неё все файлы кроме .htaccess. Проследить, чтобы IDE не изменила namespace.
4 Для Apache: в корень сайта скопировать .htaccess или внести правила в существующий .htaccess. Если используется созданная дериктория, то в правилах указать правильный путь до router.php (пр.: RewriteRule .* yourdir/router.php [L])
  Для nginx: в конфигурационный файл в секцию server добавить правило:
        if (!-e $request_filename) {
           rewrite ^(.+)$ /router.php;
        }
  Если используется созданная дериктория, то в правилах указать правильный путь до router.php:
        if (!-e $request_filename) {
           rewrite ^(.+)$ /yourdir/router.php;
        }

Хранение URL организовано в файле в виде данных JSON.
Правила из .htaccess и конфигурации nginx не будут затрагивать файлы и дериктории на сайте. Остальные адреса с домена будут вести к router.php.
Для лучшего роутинга можно в правилах .htaccess или конфигурационном файле nginx добавить маску.
